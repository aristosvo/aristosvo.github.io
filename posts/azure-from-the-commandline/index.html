Azure Context Switching from the Commandline

December 10, 2019 3-minute read

One of the subjects I wanted to touch on this blog is my journey in my new job. And with my new job a new cloud adventure started. In my previous job I was primarily focussed on IBM Cloud, AWS and a bit of Google Cloud. Although all clouds are in some way the same, there are also a lot of difficulties, hick-ups and new ideas marching around the corner.

Switching subscriptions

Because of our work for different clients, my first bash script I wrote to make work easier was a script to switch subscriptions, saz or switchazure:

#!/usr/bin/env bash

unset options i
while IFS= read -r -d $'\n' f; do
  options[i++]="$f"
done < <(az account list --query '[].{id:id, name:name}' -o tsv | cut -f 2,1 )
tLen=${#options[@]}
select opt in "${options[@]}" "Stop the script"; do
  if [ "$REPLY" -le "$tLen" ]; 
  then
    subscriptionid=`echo $opt | cut -f 1 -d " "`
    echo -e "--> Set Subscription: az account set -s ${subscriptionid}\n"
    az account set -s $subscriptionid
    
    echo "--> Subscription is:"
    az account show -o table
    break
  elif [ "$REPLY" -eq "$(($tLen+1))" ];
  then
    echo "--> Stopped the script"
    break
  else
    echo "Ho ho ho. Something went wrong. ${REPLY} is no good a selector"
    break
  fi
done

I put it here: ~/azure-script/switchazure.sh and voila!

For the people who are like me (lazy and not born from the bash shell)

alias saz='~/azure-scripts/switchazure.sh'
chmod +x '~/azure-scripts/switchazure.sh'

To be honest, I use this every day a few times a day. In the future it will probably change a bit due to the level of automation I’m planning to bring into the organisation, but for now it works like a charm.

Kubernetes switching

A simular approach worked for me, adding Kubernetes contexts. I don’t want to do the lookup myself;) After that, I use kubectx on my imported context.

#!/usr/bin/env bash

unset options i
while IFS= read -r -d $'\n' f; do
  options[i++]="$f"
done < <(az aks list --query '[].{name:name, resourcegroup:resourceGroup}' -otsv | cut -f 1,2 )
tLen=${#options[@]}
select opt in "${options[@]}" "Stop script"; do
  echo $opt
  if [ "$REPLY" -le "$tLen" ]; 
  then
    clustername=`echo $opt | cut -f 1 -d " "`
    resourcegroup=`echo $opt | cut -f 2 -d " "`
    echo -e "\n--> Get Credentials: az aks get-credentials -n ${clustername} -g ${resourcegroup}"
    az aks get-credentials -n $clustername -g $resourcegroup
    # echo -e "\n--> Contexts available are:"
    # kubectl ctx
    break
  elif [ "$REPLY" -eq "$(($tLen+1))" ];
  then
    echo -e "\n--> Script stopped"
    break
  else
    echo -e "\n--> Script stopped"
    echo "Ho ho ho. Something went wrong. ${REPLY} is no good a selector"
    break
  fi
done

I put it here: ~/azure-script/getaksctx.sh and voila!

For the people who are like me:

alias gac='~/azure-scripts/getaksctx.sh'
chmod +x '~/azure-scripts/getaksctx.sh'
